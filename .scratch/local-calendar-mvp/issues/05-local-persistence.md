# 05 — 持久化本地日历数据

**What to build:** 新增和删除后的事项保存到当前设备，重新加载应用后仍能看到正确的本地数据；存储边界保持可替换，后续可以迁移到数据库而不改变用户可见行为。

**Blocked by:** 03 — 补齐事项时间规则、排序和日期标记；04 — 删除事项并二次确认

**Status:** resolved

- [x] 新增事项后，本地存储保存事项数据。
- [x] 重新加载应用状态后，之前新增的事项仍显示在正确日期。
- [x] 删除事项后，本地存储同步移除该事项。
- [x] 重新加载应用状态后，被删除的事项不会重新出现。
- [x] 本地数据只保存在当前设备，不接入账号、云同步或系统日历。
- [x] 存储实现通过窄边界接入，用户界面不依赖具体存储机制。
- [x] 相关测试使用可替换的本地存储边界或测试存储实现验证持久化行为。
