# 05 — 日程页视觉系统迁移

**What to build:** 将日程页迁移到应用级设计系统，统一页面标题、空状态、日期分组、事项列表项、删除入口和删除确认。完成后日程页更清晰地按时间线呈现全部本地事项，且分组排序、空状态跳转和删除同步行为全部保持不变。

**Blocked by:** 01 — 应用级主题 token 骨架；02 — 共享 UI 组件首批落地；03 — 应用框架和底部导航视觉统一

**Status:** resolved

- [x] 日程页使用统一页面容器、页面标题和说明文字。
- [x] 没有事项时使用共享空状态，并保留去日历页添加入口。
- [x] 点击空状态入口后切换到日历页，但不自动打开新增事项表单。
- [x] 日期分组使用统一内容卡片、日期标题和组内间距。
- [x] 事项列表项使用统一标题、时间、颜色标记和删除入口样式。
- [x] 删除事项确认使用共享确认弹窗样式。
- [x] 事项仍按日期从早到晚分组，同一日期内按事项时间升序排列。
- [x] 日程页删除事项后，日历页当天列表和日期标记同步更新。
- [x] 相关 widget 测试覆盖分组排序、空状态跳转、删除取消和删除确认。

## Answer

已将日程页迁移到 `AppPageContainer`、`AppPageTitle`、`AppEmptyState` 和 `AppContentCard`。日期分组、事项标题、时间说明、颜色标记、删除入口和组内间距均读取应用级 theme token；删除事项继续使用共享确认弹窗。

保留原有空状态跳转、日期分组排序、同日按开始时间排序、删除取消和删除确认后的跨页面同步行为。新增 `test/app_theme_test.dart` 断言日程页列表消费应用设计 token，既有 `test/widget_test.dart` 覆盖分组排序、空状态跳转和删除同步。
