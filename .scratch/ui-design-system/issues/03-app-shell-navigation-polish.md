# 03 — 应用框架和底部导航视觉统一

**What to build:** 统一四个功能页面共享的应用框架视觉，包括背景、页面外边距、安全区、底部导航、浮动新增按钮和导航选中态。完成后用户在四个功能页面之间切换时能感到这是同一套产品壳，且原有 tab 切换和页面状态保留行为不变。

**Blocked by:** 01 — 应用级主题 token 骨架；02 — 共享 UI 组件首批落地

**Status:** resolved

- [x] 四个功能页面共享统一背景和页面容器规则。
- [x] 页面外边距、最大宽度、安全区和底部导航避让由统一规则控制。
- [x] 底部导航使用统一图标尺寸、文字样式、触控区域和选中态。
- [x] 底部导航切换后当前选中功能页面高亮正确。
- [x] 切换离开日历页再返回时，日历页当前显示月份和选中日期保持不变。
- [x] 浮动新增事项按钮使用统一主操作样式，且只在日历页显示。
- [x] 移动端窄屏下底部导航文字和图标不溢出、不重叠。
- [x] 相关 widget 测试覆盖底部导航切换、选中态和功能页面状态保留。

## Answer

已抽出应用框架共享尺寸常量，`AppPageContainer` 使用统一最大宽度和动态底部导航避让。底部导航迁移为主题驱动的 `NavigationBar`，共享导航高度、最大宽度、表面色、边框和阴影，并保留当前选中功能页面状态和日历页状态。日历页浮动新增事项按钮改为读取主操作色、前景色、阴影和统一尺寸，且仍只在日历页显示。

新增 widget 测试覆盖底部导航主题化、选中态切换、日历页状态保留、浮动新增按钮 token 使用，以及 320px 窄屏下导航文字和图标不溢出。
