# 完整功能页面规格

标签：ready-for-agent

## Problem Statement

当前本地日历已经能围绕选中日期创建、查看、删除和持久化事项，但应用仍然只有日历页是真实功能页面。底部导航中的日程、待办和我的仍没有对应体验，日历页也固定在演示月份，用户无法跨月份规划。本地日历需要从单页 MVP 发展成一个更完整的四功能页面应用，让用户能在同一份本地数据上查看事项、管理待办、查看统计并清理本地数据。

## Solution

将底部导航升级为四个可进入的功能页面：日历页、日程页、待办页和我的页。日历页保留现有视觉和事项创建能力，并增加上个月/下个月的月份切换；日程页跨日期展示全部事项，支持删除事项，并在空状态引导用户去日历页添加；待办页管理独立于事项的待办项，支持新增、完成切换和删除确认；我的页展示应用信息、本地数据统计，并提供二次确认后的清空本地数据入口。四个功能页面共享同一份本地数据，底部导航切换时保留各功能页面状态。

## User Stories

1. 作为本地日历用户，我想点击底部导航的日历入口，以便回到月份网格和当天事项视图。
2. 作为本地日历用户，我想点击底部导航的日程入口，以便跨日期查看全部事项。
3. 作为本地日历用户，我想点击底部导航的待办入口，以便管理没有固定事项时间的待办项。
4. 作为本地日历用户，我想点击底部导航的我的入口，以便查看本地应用信息和本地数据入口。
5. 作为本地日历用户，我想切换底部导航后页面状态仍保留，以便回到日历页时仍看到之前选中的日期。
6. 作为本地日历用户，我想日历页显示当前显示月份，以便知道自己正在查看哪个月份。
7. 作为本地日历用户，我想在日历页切换到上个月，以便查看和规划过去月份的事项。
8. 作为本地日历用户，我想在日历页切换到下个月，以便查看和规划未来月份的事项。
9. 作为本地日历用户，我想月份切换后优先选中相同日序，以便保持连续的日期上下文。
10. 作为本地日历用户，我想当目标月份没有原日序时自动选中目标月份最后一天，以便选中日期始终有效。
11. 作为本地日历用户，我想月份网格反映当前显示月份的真实日期，以便不再被固定演示月份限制。
12. 作为本地日历用户，我想跨月份创建事项，以便把安排保存到不同月份。
13. 作为本地日历用户，我想新增事项表单仍使用当前选中日期，以便创建入口保持简单清晰。
14. 作为本地日历用户，我想日历页只显示选中日期的事项，以便当天视图不混入其他日期。
15. 作为本地日历用户，我想日历页月份标记反映当前显示月份内有事项的日期，以便快速扫出哪些天有安排。
16. 作为本地日历用户，我想日程页显示全部本地事项，以便不用逐天点选也能查看安排。
17. 作为本地日历用户，我想日程页按日期从早到晚分组，以便按时间线阅读事项。
18. 作为本地日历用户，我想日程页每个日期组内按事项时间升序排列，以便一天内安排顺序清楚。
19. 作为本地日历用户，我想从日程页删除事项，以便跨日期管理已有事项。
20. 作为本地日历用户，我想日程页删除事项前二次确认，以便误触不会立即移除本地数据。
21. 作为本地日历用户，我想日程页删除事项后日历页同步更新，以便不同功能页面看到同一份本地数据。
22. 作为本地日历用户，我想日程页没有事项时看到空状态，以便知道当前没有本地事项。
23. 作为本地日历用户，我想日程页空状态提供去日历页添加的入口，以便知道下一步在哪里创建事项。
24. 作为本地日历用户，我想待办页显示全部待办项，以便集中管理轻量清单。
25. 作为本地日历用户，我想在待办页新增待办项，以便记录不占用日历时间的事情。
26. 作为本地日历用户，我想新增待办项时填写标题，以便识别待办内容。
27. 作为本地日历用户，我想新增待办项时可以填写备注，以便保留轻量上下文。
28. 作为本地日历用户，我想标题缺失时不能保存待办项，以便待办列表不出现空内容。
29. 作为本地日历用户，我想切换待办项完成状态，以便区分已完成和未完成。
30. 作为本地日历用户，我想取消完成待办项，以便修正误操作。
31. 作为本地日历用户，我想未完成待办项显示在已完成待办项之前，以便优先看到还要处理的内容。
32. 作为本地日历用户，我想同组待办项按创建时间倒序显示，以便最近创建的待办更容易看到。
33. 作为本地日历用户，我想删除待办项，以便移除不需要的清单内容。
34. 作为本地日历用户，我想删除待办项前二次确认，以便误触不会立即移除本地数据。
35. 作为本地日历用户，我想待办页没有待办项时看到空状态，以便知道当前没有待办。
36. 作为本地日历用户，我想待办页空状态提供新增待办入口，以便可以直接开始创建。
37. 作为本地日历用户，我想关闭并重新打开应用后待办项仍然存在，以便待办页是本地持久功能。
38. 作为本地日历用户，我想我的页显示应用名称和简要说明，以便理解这是本地日历应用。
39. 作为本地日历用户，我想我的页显示事项数量，以便知道本地保存了多少事项。
40. 作为本地日历用户，我想我的页显示待办项数量，以便知道本地保存了多少待办。
41. 作为本地日历用户，我想我的页提供清空本地数据入口，以便需要时移除当前设备内的全部事项和待办项。
42. 作为本地日历用户，我想清空本地数据前看到二次确认，以便不会误删全部本地数据。
43. 作为本地日历用户，我想取消清空确认后全部本地数据保持不变，以便可以安全返回。
44. 作为本地日历用户，我想确认清空后事项和待办项全部消失，以便本地数据真正被清理。
45. 作为本地日历用户，我想清空后仍停留在当前功能页面，以便不会因为清理数据被突然切走。
46. 作为本地日历用户，我想清空后日历页选中日期保持不变，以便页面状态不因数据清理被重置。
47. 作为本地日历用户，我想清空后日程页和待办页显示空状态，以便数据变化可见。
48. 作为本地日历用户，我想清空后我的页统计变成 0，以便确认本地数据已经清空。
49. 作为本地日历用户，我不想在这轮看到账号、登录、云同步、会员或备份入口，以便应用仍保持本地使用边界。
50. 作为本地日历用户，我不想待办项混入事项列表，以便待办和事项的语义保持清楚。
51. 作为本地日历用户，我不想日程页提供新增事项表单，以便新增事项仍从日历页当前选中日期开始。
52. 作为本地日历用户，我不想在这轮使用年份/月选择器或跳转今天，以便月份切换保持简单。

## Implementation Decisions

- 底部导航从视觉壳升级为功能页面导航，包含日历页、日程页、待办页和我的页。
- 四个功能页面共享同一份本地数据，页面之间不能各自维护互不一致的存储状态。
- 底部导航切换时保留功能页面状态，至少包括日历页的当前显示月份和选中日期。
- 日历页保留现有事项创建、当天事项列表、日期标记、删除确认和本地持久化能力。
- 日历页新增上个月和下个月切换，不引入年份选择器、月份选择器或跳转今天。
- 月份切换后优先选中目标月份相同日序；目标月份没有该日序时选中目标月份最后一天。
- 新增事项表单继续使用当前选中日期，不提供表单内改日期能力。
- 日程页显示全部本地事项，按日期从早到晚分组，组内按事项时间升序排列。
- 日程页支持删除事项，并复用事项删除前二次确认的行为。
- 日程页没有事项时显示空状态，并提供跳转到日历页的入口；跳转不自动打开新增表单。
- 待办项是独立于事项的本地数据，包含标题、是否完成、创建时间和可选备注。
- 待办项不包含截止日期、优先级、分类、提醒、重复或事项时间。
- 待办页是待办项主创建入口，支持新增待办项、切换完成状态和删除待办项。
- 待办项标题必填；备注可选。
- 待办页未完成待办项显示在已完成待办项之前，同组内按创建时间倒序显示。
- 删除待办项前必须二次确认。
- 待办页没有待办项时显示空状态，并提供新增待办入口。
- 我的页显示应用名称、简要说明、本地事项数量和本地待办项数量。
- 我的页提供清空本地数据入口，清空前必须二次确认。
- 清空本地数据会删除当前设备内全部事项和待办项。
- 清空本地数据后保持当前功能页面和功能页面状态不变，数据区域自然显示空状态或零统计。
- 本轮继续使用现有应用名“日历”，不做品牌、启动页、图标或产品命名升级。
- 本轮不接入账号、云同步、系统日历、会员、备份或通知。
- 本轮需要把现有事项存储边界扩展为应用级本地数据边界，支持事项和待办项的统一读取、保存、统计和清空。
- 存储实现仍保持轻量 JSON/键值边界，不引入数据库；后续迁移数据库不应改变用户可见行为。

## Testing Decisions

- 测试应关注用户可观察行为，不断言私有 widget 或私有状态实现。
- 主要测试切入点是最高层 `CalendarApp` widget，因为它能覆盖底部导航、页面状态保留、共享本地数据、月份切换、日程页、待办页和我的页之间的用户可见联动。
- 本地数据测试切入点是应用级本地数据存储边界。内存实现用于 widget 测试；文件实现用于 JSON round-trip 和清空行为测试。
- 现有事项创建、删除、排序、日期标记和持久化 widget 测试应继续保留，并随着四功能页面结构调整到最高有效 seam。
- 增加底部导航切换测试，验证可以进入日程页、待办页和我的页。
- 增加功能页面状态保留测试，验证日历页选中日期或当前显示月份在 tab 切换后不重置。
- 增加月份切换测试，覆盖上个月、下个月、相同日序保留和月末回退。
- 增加跨月份事项测试，验证新增事项、日期标记和当天列表使用当前显示月份和选中日期。
- 增加日程页测试，覆盖事项分组排序、删除确认、删除后日历页同步更新和空状态去日历页入口。
- 增加待办页测试，覆盖新增、标题必填、完成切换、取消完成、排序、删除取消和确认。
- 增加我的页测试，覆盖事项/待办项统计、清空确认取消、清空确认成功以及清空后页面状态保留。
- 增加本地数据持久化测试，验证事项和待办项关闭重开后仍能加载，清空后不会重新出现。
- 修改代码后按项目规则检查 UTF-8/中文乱码。

## Out of Scope

- 编辑已有事项。
- 编辑已有待办项。
- 待办项截止日期、优先级、分类、提醒、重复。
- 将待办项安排为事项，或事项和待办项之间的转换。
- 年份选择器、月份选择器和跳转今天。
- 账号、登录、云同步、备份、会员或多设备一致性。
- 读取或写入系统日历。
- 提醒和通知。
- 事项地点、参与人、附件、优先级和标签。
- 全天事项和跨天事项。
- 搜索、筛选和分类管理。
- 品牌升级、启动页、应用图标和产品名重命名。
- 迁移到 SQLite、Drift 或其他数据库。

## Further Notes

本规格建立在已完成的本地日历 MVP 之上。上一轮 `local-calendar-mvp` tickets 已解决事项创建、当天列表、时间规则、删除确认和本地持久化；本轮应尽量复用这些行为，并把它们纳入四功能页面结构。实现前应注意当前仓库可能已有未提交的 `CONTEXT.md` 领域文档改动，不要回退。
