# 06 — 待办页完成状态和删除确认

**What to build:** 用户可以在待办页切换待办项完成状态，也可以删除待办项。未完成待办项显示在已完成待办项之前，同组内按创建时间倒序显示；删除待办项前需要二次确认。

**Blocked by:** 05 — 待办页新增和持久化待办项

**Status:** ready-for-agent

- [ ] 每个待办项显示完成状态。
- [ ] 用户可以将未完成待办项标记为已完成。
- [ ] 用户可以将已完成待办项恢复为未完成。
- [ ] 完成状态变化会保存到本地数据。
- [ ] 重新打开应用后完成状态保持不变。
- [ ] 未完成待办项显示在已完成待办项之前。
- [ ] 同组待办项按创建时间倒序显示。
- [ ] 每个待办项提供删除入口。
- [ ] 点击删除入口后显示二次确认弹窗。
- [ ] 取消删除后待办项保持不变。
- [ ] 确认删除后待办项从待办页消失。
- [ ] 删除最后一个待办项后，待办页显示空状态。
- [ ] 相关 widget 测试覆盖完成、取消完成、排序、删除取消和删除确认。
