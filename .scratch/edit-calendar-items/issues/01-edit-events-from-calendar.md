# 01 — 日历页编辑事项

**What to build:** 用户可以从日历页当天事项列表打开预填的编辑表单，修改已有事项的标题、开始时间、结束时间、备注和颜色；保存后原事项被替换、当天列表重新排序并持久化，取消时原事项保持不变。

**Blocked by:** None — can start immediately

**Status:** resolved

- [x] 日历页每条事项提供明确可见的编辑入口，且不依赖长按或点击整行。
- [x] 编辑入口具有包含事项语义的可访问名称。
- [x] 编辑表单标题和提交语义明确表示正在编辑事项。
- [x] 编辑表单预填原事项的标题、开始时间、结束时间和备注。
- [x] 编辑表单预选原事项颜色。
- [x] 编辑表单显示原事项所属日期，但不允许修改日期。
- [x] 用户可以修改标题、开始时间、结束时间、备注和颜色。
- [x] 标题、开始时间和结束时间继续执行新增事项时的必填与格式校验。
- [x] 结束时间不晚于开始时间时不能保存，并显示现有时间规则的校验反馈。
- [x] 用户修正字段后，对应错误提示及时消失，未触碰字段不会在提交前提前显示错误。
- [x] 保存后替换原事项，不新增副本，事项总数保持不变。
- [x] 修改开始时间后，当天事项按新的开始时间重新排序。
- [x] 保存后编辑结果写入本地数据，重新构建或重新加载应用后仍然存在。
- [x] 取消或关闭编辑表单后，原事项及本地数据保持不变。
- [x] 原有新增事项、删除事项和日期标记行为继续正常工作。
- [x] 相关用户可观察行为由最高层应用 widget 测试覆盖。

## Answer

已实现并通过事项编辑、校验、排序、取消、持久化与回归测试。
