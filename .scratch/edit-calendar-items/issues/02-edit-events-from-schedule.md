# 02 — 日程页编辑事项

**What to build:** 用户可以从日程页跨日期事项列表打开同一套事项编辑表单；保存后日程页和日历页同步显示修改结果，事项数量和所属日期不变，并继续持久化到本地数据。

**Blocked by:** 01 — 日历页编辑事项

**Status:** ready-for-agent

- [ ] 日程页每条事项提供明确可见的编辑入口，且不依赖长按或点击整行。
- [ ] 日程页编辑入口具有包含事项语义的可访问名称。
- [ ] 从日程页打开的编辑表单预填事项原有字段和颜色。
- [ ] 用户可以修改标题、开始时间、结束时间、备注和颜色，但不能修改所属日期。
- [ ] 日程页编辑复用日历页已经建立的表单模式和校验规则，不形成平行编辑逻辑。
- [ ] 保存后日程页立即显示修改结果，并按修改后的时间保持正确分组和组内排序。
- [ ] 从日程页编辑后，返回日历页可看到同一事项的更新结果。
- [ ] 保存后替换原事项，不新增副本，事项总数和“我的”页事项统计保持不变。
- [ ] 取消或关闭编辑表单后，日程页和日历页中的原事项保持不变。
- [ ] 编辑结果写入本地数据，重新构建或重新加载应用后仍然存在。
- [ ] 日程页原有删除确认、空状态和去日历页入口继续正常工作。
- [ ] 跨页面同步和持久化行为由最高层应用 widget 测试覆盖。
